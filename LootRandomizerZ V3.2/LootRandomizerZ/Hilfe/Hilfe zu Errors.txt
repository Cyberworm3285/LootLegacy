Die unter Umständen beim Validieren entstehenden Err0rs bedeuten folgendes:
Ungültige Anzahl-Konfig!(E00)	->	hinter der ersten Klammer ( '(' ) ist keine Zahl
Ungültige Anzahl-Konfig!(E01)	->	vor dem trennenden Komma ( ',' ) ist keine Zahl
Ungültige Anzahl-Konfig!(E02)	->	nach dem trennenden Komma ( ',' ) ist keine Zahl
Ungültige Anzahl-Konfig!(E03)	-> 	vor der zweiten Klammer ( ')' ) ist keine Zahl
Ungültige Wert-Konfig!(E04) 	-> 	hinter der ersten Klammer ( '{' ) ist keine Zahl
Ungültige Wert-Konfig!(E05)	->	vor dem trennenden Strich ( '-' ) ist keine Zahl
Ungültige Wert-Konfig!(E06)	->	hinter dem trennenden Strich ( '-' ) ist keine Zahl
Ungültige Wert-Konfig!(E07)	-> 	vor der zweiten Klammer ( '}' ) ist keine Zahl
Ungültige Attribut-Konfig(08)!	->	hinter der Raute ( '#' ) ist keine Zahl
Ungültige Gebiets-Konfig!(E09)	->	hinter dem Schrägstrich ( '\' ) ist keine Zahl
Ungültige Gebiets-konfig!(E10)	->	der Gebiets-Tag über der Nummer des höchsten Gebiets
Ungültige Gebiets-konfig!(E11)	-> 	der Gebietstag ist unter 1
I. ist Ruestung und Waffe!(E12)	->	Das Item ist sowohl mit '$' und '~' getagged
Deklariert, aber kein Wert!(W00)->	[Warnung] Das Item ist zwar als Waffe/Rüstung getagged, 
					bekommt aber keinen Wert zugewiesen (nicht essentiell)
	
Gebietsdatei ist leer!(W01)	->	[Warnung] Die Gebietsdatei ist leer und deshalb ist
					die Hilfsfunktion deaktiviert
